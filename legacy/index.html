THINKING FRAMEWORKS

核心思考

这些不是独立项目，而是我在文章中反复打磨的几个核心框架。它们构成了我观察 AI 系统的坐标系。

Agent Architecture

Agent = Model + Harness

模型像一匹能力很强的马，但没有马具、缰绳和骑手，它不会自动奔向业务目标。Harness 是 Agent 外侧的工程支架：环境、工具链、权限、状态、验证、观测与反馈。

  • Prompt → Context → Harness
  • 执行层
  • 验证闭环

Human-AI Collaboration

HITL / HOTL / HOOTL 控制谱系

不是"人重要不重要"，而是"人的判断发生在什么时间点，以什么方式影响系统"。三种模式构成一条连续谱，帮助产品经理决定每一步 AI 动作的控制策略。

  • 预防 · 检测 · 自动化
  • 控制权分配
  • 可观测性

Engineering Philosophy

工程过程的 Token 化边界

AI 的能力边界不取决于模型本身，而取决于工程过程可被 token 化的范围。多工作树低冲突合并是执行层问题，工程 token 化是信息层问题，两者合在一起才是生产力基础。

  • 多工作树
  • 低冲突合并
  • 验证独立性
